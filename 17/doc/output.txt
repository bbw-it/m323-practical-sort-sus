Natural-Order:
Java verwendet TimSort um Daten zu sortieren. Integer werden nach grsse sortiert und Strings werden Alphabetisch sortiert.
Dabei muss das Interface Comparable implementiert sein, somit kann mit compareTo(T) die klasse sortiert werden.
Desswegen ist es manchmal mhsam wenn man z.B. bei Strings: [1,5,9,11,12] denn sortiert es dies so: [1,11,12,5,9].
Das ganze kann man mit Zero padding "fixen".

Reverse-Order:
Es macht genau das gleiche wie bei Natural-Order aber das ergebniss wird negiert.

Wie sortiert Java:
Es nutzt TimSort, eine mischung von MergeSort und InsertionSort.
Ich dachte immer es wre bubbleSort aber mit O(n) ist es viel langsamer als TimSort welches von O(n) zu O(n log n) geht.

Was macht Java genau beim Sortieren:
Es Prft ob die Liste <= 1 ist, wenn ja ist das Sortieren direkt fertig.
Danach fhrt es TimSort aus. Es wird die Methode compareTo(T) oder den Comperator ausgefhrt. 
Die Sortierung verwendet die werte die bei compareTo oder Comparator zurck kommen, es wird nichts selbst von Java convertiert.
